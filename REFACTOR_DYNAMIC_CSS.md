# CSS 动态视口缩放重构状态

> **最后更新**: 2026-05-01
> **基准窗口**: 1912 * 948 (`innerWidth * innerHeight`)
> **目标**: 用连续缩放 token 和级联字号模型替代固定像素与尺寸断点补丁。

---

## 1. 当前原则

- 基准窗口下的视觉尺寸必须保持原样；`clamp()` 中间值按 1912 * 948 换算。
- 纵向尺寸和字号优先跟随视觉高度，横向尺寸优先跟随视觉宽度。
- 内距、间距和小控件偏移优先用 `em` 或从已有尺寸 token 推导。
- `html.force-landscape` 只负责轴向互换或少量全局旋转结构，不新增组件级覆盖选择器。
- 删除旧定义时优先删掉已被动态 token 覆盖的固定值、重复选择器和尺寸断点补丁。
- 结构切换断点可以保留，但必须是布局形态变化，不应只改固定尺寸。

更完整的反模式清单见 `ARCHITECTURE.md` 0.12，级联模型说明见 0.13。

## 4. 保留的非尺寸断点

以下断点不是旧尺寸补丁，暂时保留：

- `style/media/main_perspective.css` 的 `768px` 断点：切换当前角色信息的换行后参数。
- `style/modals/base.css` 的 `768px` 断点：词元弹窗从双列编辑切换为单列。
- `style/skill_copy.css` 的 `640px` 断点：复制按钮在窄屏下从行外悬浮改为可占位显示，属于交互策略。
- `prefers-reduced-motion`、`prefers-contrast`、`hover`、`pointer` 等能力查询：属于可访问性或输入设备策略。

后续若要删除这些断点，必须先替换成等效的结构策略，例如容器查询或不依赖宽度的流式布局。

## 7. 后续原则

动态缩放主线已完成。后续不再按“大迁移”推进，而是在真实内容验收中处理局部问题：

- 如果发现标准窗口尺寸漂移，优先检查 token 的基准换算。
- 如果发现小窗口拥挤，优先调现有 token 参数，不新增覆盖选择器。
- 如果发现旧固定尺寸，先判断它是否是边框、滚动条、动画关键帧或无关装饰；只有影响布局或比例时才迁移。
- 如果发现 `transition: all`，优先收窄到实际变化的属性。
