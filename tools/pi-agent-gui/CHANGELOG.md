# Pi Agent GUI 更新日志

这个日志只记录 `tools/pi-agent-gui/` 工具本身的变化，避免混入网站面向玩家的 `base/announcements.json`。

## 2026-06-21

- 修复：编辑旧消息但尚未重发时，后台消息刷新不再把已撤回的原历史内容重复带回消息区。
- 修复：Pi 运行跟踪在 Activity 面板显示 server、runtime、request、response 和 stderr/stdout 阶段；运行时尚未输出时也会显示 GUI 已连接的等待状态，避免一直停在“暂无输出”。
- 优化：Pi 对话进行中页面更流畅。
- 优化：Activity 面板新增固定的待确认文件编辑列表，显示每个文件的状态、增删数、确认/取消按钮，并可打开仅限本次编辑的差异高亮。
- 优化：编辑重发进入草稿时只隐藏后续消息，发送时才撤回绑定的文件编辑 checkpoint；取消编辑会恢复当前视图，避免点击编辑就不可逆地改动文件。
- 优化：编辑步骤里的文件路径保持可点击路径标记；命中本次编辑 checkpoint 时会复用 scoped diff 入口，未命中时也不会退成纯文本。
- 修复：确认或取消待确认文件编辑后，会同步收起对应的本次编辑 diff 预览，避免列表已消失但预览仍显示未确认状态。

## 2026-06-19

- 新增：文件编辑建立会话内检查点，消息区会显示待确认文件编辑，可确认保留或取消并按快照恢复；从旧用户消息回退重发时会同步撤回该检查点之后的文件编辑。
- 优化：编辑类步骤保留更多工具 payload，支持从 diff、patch、stats、additions/deletions 等多种形态解析行数增减，并在步骤状态中显示 `+N / -M`。
- 新增：用户消息增加“检查点”和“编辑重发”入口，可回到旧消息位置撤回界面中的后续内容并重新发送；当前实现不直接改写 Pi 原生 session 文件，避免伪造未暴露的运行时回滚能力。
- 优化：文件预览高亮复用一次性行号索引生成概览标记，长文件改为按可见行挂载正文高亮节点，保留语法高亮和右侧概览，同时减少打开时一次性插入大量 DOM 的卡顿。

## 2026-06-18

- 新增：会话可查看历史消息、消息时间、运行状态和工具活动，刷新页面重新加载会话时会默认回到最新消息。
- 修复：停止按钮会真正停止共享 Pi 进程，停止进程不再扰动消息列表。
- 优化：补齐上下文同步、模型切换、发送失败、活动去重、复制消息不跳到底部、移除异常的重发入口和减少消息列表刷新闪烁。
- 新增：主对话区实时显示正在思考、最终响应、归入 Agent 消息的折叠工具步骤、连续文本步骤过程反馈、可折叠的单条步骤、步骤动作名称、运行中状态与 IDE 工具名称语义化。
- 优化：步骤分类覆盖读取、搜索、编辑、运行、检查和页面验证等操作，活动页可搜索查看步骤类型参考，工具参数支持结构化展示。
- 优化：回复支持基础 Markdown 格式，标题、引用块和正文颜色层级会按语义显示。
- 优化：上下文区域可直接添加、移除和拖入文件或文件夹，并在项目树中标出已加入上下文的条目。
- 新增：消息正文和步骤标题中的项目路径会显示为可点击的文件或文件夹标记，可跳转到预览或项目树。
- 优化：路径图标间距与项目列表保持一致；刷新后输入框保留最小可见高度，上下文内容已足够显示时不再继续拉伸留白。
- 优化：消息和步骤里的路径标记垂直居中显示，项目树截断时仍会把消息里的文件、显式文件夹路径和可唯一定位的裸文件名识别成可点击标记，文件夹作为上下文时会和文件一样在项目树高亮，编辑类步骤可在状态元信息中展示 diff 增删统计，运行命令类步骤标题不再直接塞入完整命令。
- 修复：运行状态短暂缺省时不再报错。

## 2026-06-17

- 新增：Sessions 面板可列出本机 Pi session 文件，选择下次启动 runtime 使用的默认、继续上次、指定 session、指定 session-id 或无会话模式；运行中保留 Pi RPC 新建会话，不伪造未验证的切换能力。
- 优化：输入区上下文从底层范围/checkbox 中间形态改为“已添加上下文”chip 列表；可通过添加菜单加入当前文件、当前 Diff、Git 变更和未保存状态，也可把项目树文件拖入输入区形成可移除文件 chip。
- 优化：`PLAN.md` 补充 Pi Agent GUI 设计门禁，要求先定义用户对象、agent owner 和显示开关，再进入实现，减少把中间权限机制误做成交互设计。
- 新增：GUI server 接入 Pi 原生 session 启动参数，可通过环境变量传递 `--session-id`、`--session`、`--session-dir`、`--continue`、`--resume` 和 `--no-session`，Runtime 面板显示当前 session owner。
- 新增：IDE bridge 补齐 `read_ide_file`、`read_ide_diff`、`list_ide_changes` 和 `get_unsaved_buffers` 工具，并由 server 端按当前上下文范围统一做读取门禁。
- 新增：输入区增加 IDE 上下文范围选择（全部、当前文件、当前 Diff、Git 变更、无），`get_ide_context` 只返回所选范围内的状态。
- 优化：撤回浏览器侧消息缓存，后续会话恢复改接 Pi 原生 session 或 GUI server 侧 session store，避免桌面与手机各自形成历史来源。
- 新增：会话消息补齐复制、用户消息重发、Assistant 当前模型标识、思考过程折叠块和工具调用/工具结果折叠块，更接近 IDE 内 Copilot 面板的阅读体验。
- 优化：Git 变更列表显示每个文件的 `+/-` 增删统计，便于在手机端快速判断变更规模。
- 修复：gpt5.5 中转默认 Base URL 改为 `https://ai.lupoapi.com/v1`。VS Code custom endpoint 配置可填站点根地址，但 Pi/OpenAI SDK 会把 `baseUrl` 直接与 `/chat/completions` 组合，因此 GUI 侧默认值必须使用 API 根路径。
- 新增：Pi runtime 启动时自动加载 `extension/ide-bridge.js`，首个 `get_ide_context` 工具可读取 GUI 当前项目、活动文件、活动 diff、未保存状态和 Git 变更摘要。
- 新增：GUI server 增加 IDE bridge 状态接口，浏览器在打开文件、查看 diff、保存和编辑时同步当前 IDE 状态，供 Pi extension 按需读取。
- 修复：模型列表在 runtime 未启动时也会从 `models.json` 读取，保存或删除 Provider 后立即刷新，避免未配置模型阶段无法继续试用配置页。
- 优化：Provider 配置里的 API Key 字段明确支持直接粘贴 Key 或填写环境变量引用，明文保存后不在页面回显，降低首次配置成本。
- 优化：Provider 表单默认填入 `Context=128000` 和 `Max tokens=16000`，新建自定义中转模型时不再需要手动补这两个常用值。
- 优化：Provider 兼容性选项的标签和说明改为 Pi `models.json` 源码语义，明确对应 `reasoning`、`supportsDeveloperRole` 和 `supportsReasoningEffort`。
- 优化：Provider 表单默认带入 `gpt-5.5` 中转测试参数（不含 API Key），方便只粘贴 Key 后直接保存试跑。
- 修复：Provider 保存后刷新页面会从 `models.json` 回填已保存配置；明文 Key 仍不回显，但再次保存其它字段时会保留原 Key。
- 修复：模型面板现在会在 runtime 未启动时也读取 `models.json`，避免后端已有模型但下拉框仍显示为空。
- 修复：发送消息时不再把 IDE 引用和 Git 变更拼进正文；这些状态仅通过 IDE bridge/tool 供 agent 按需读取。
- 优化：同步更新 README、计划和静态界面文案，移除旧的 prompt fallback 表述，避免后续实现回到正文拼接路线。
- 优化：将输入区开关和帮助说明的内部 owner 从 `includeContext` 收敛为 `showIdeState`，明确它只控制人类预览，不控制发送正文。
- 修复：`start-tailscale.bat` 启动前会清理旧的 Pi Agent GUI server，避免旧的 `127.0.0.1:3002` 进程占用端口导致手机端访问不存在。
- 新增：补充 `PLAN.md` 项目计划，明确 Pi Agent GUI 后续要从 RPC 包装层升级为 Pi-native IDE bridge，把项目、Git、编辑器状态和上下文选择沉淀为 extension/custom tool 能力。

## 2026-06-16

- 新增：输入区曾增加上下文开关和预览；当前版本已调整为“预览 IDE 状态”，仅供人类查看，不再随消息发送。
- 修复：上下文状态行改为贴合输入区的轻量样式，展开预览改为不撑高输入区的浮层，避免新增 UI 像独立卡片并破坏小屏布局。

## 2026-06-15

- 优化：文件预览编辑增加未保存、保存中、保存失败状态，保存成功后自动刷新 Git 变更列表并同步当前文件高亮。
- 优化：移动端工作台补齐 VS Code 式分隔条，支持项目区/聊天区、预览区/对话区、对话区/输入区之间按当前内容动态伸缩。
- 修复：运行面板侧栏分隔条不再被标题、按钮或横向预览分隔条的滚动宽度反向推宽，左右拖动方向与实际指针一致。
- 修复：手机竖屏分隔条保留 12px 触摸热区，但只显示 1px 中线，避免中间出现突兀黑色横条。
- 优化：手机端默认压缩项目区和输入区占比，分隔条改为 1px 布局轨道加 12px 触摸热区，给聊天和预览释放更多空间。
- 修复：手机浏览器按当前可见视口计算工作台高度，避免地址栏/工具栏导致页面比屏幕多出约 100px。
- 修复：手机端预览/对话分隔条现在会实际调整预览高度，并扩大触摸命中区以便手指拖动。
- 优化：低高度横屏小屏改为左右分区，避免预览、对话和输入区互相挤压重叠。
- 修复：手机端项目状态文本保持可见，预览标题图标不再换行，拖动预览时会保留对话区可用高度。
- 优化：小屏下压缩配置表单、变更列表和代码预览的行距与内距，让同一屏可见更多文件和代码内容。

## 2026-06-14

- 修复：预览概览标记按全文件范围抽样显示，避免大文件后半段没有高亮。
- 优化：变更列表增加跳转项目文件入口，可以从 Git 变更直接打开并定位对应文件。

## 2026-06-13

- 新增：完成 Pi 双端 GUI 原型，电脑端和手机端浏览器连接同一个 Pi RPC runtime。
- 新增：支持通过 `start-tailscale.bat` 一键以 Tailscale 方式启动，默认可用 `0.0.0.0:3002` 访问，并支持无 token approve 模式。
- 新增：加入运行状态、会话消息、工具活动、发送模式、Abort、Steer、Follow-up 等共享 agent 控制能力。
- 新增：加入模型 Provider 配置界面，可在浏览器中维护 provider、base URL、API key 引用、模型 ID、上下文和推理相关开关。
- 新增：加入项目文件树、项目搜索、Git 变更列表、文件读取、diff 预览和文件保存能力，让手机端不再只是 chat 入口。
- 优化：界面改为中英双语，可在页面内切换主要控件、说明和状态文案。
- 优化：增加固定说明面板，悬浮、点击或聚焦控件时展示当前位置的功能解释。
- 优化：调整为移动端和桌面端自适应工作台，尽量在固定视口中完成项目、变更、预览和对话操作。
- 优化：预览区增加文件类型图标、语法高亮、搜索命中高亮和 VS Code 风格概览条。
- 优化：diff 预览保留变更增删语义，同时显示代码语法高亮。
- 优化：普通文件编辑复用高亮预览层，避免切到独立编辑框时出现缩放跳变和高亮丢失。
- 优化：概览条拆分为代码结构轨和变更/搜索/错误轨；普通文本行、单行和多行变更标记的显示更加统一。
